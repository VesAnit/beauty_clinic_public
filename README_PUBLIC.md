# AI-ассистент для клиники эстетической медицины

## Публичная ограниченная версия

Это публичная ограниченная версия реализации агента-администратора для бьюти-клиники. Данный репозиторий содержит демонстрационный ноутбук, показывающий базовую концепцию работы системы поиска процедур через RAG (Retrieval-Augmented Generation).

## Бизнес-идея

**Проблема:** Клиники эстетической медицины ежедневно получают множество запросов от клиентов о процедурах, их стоимости, длительности, специалистах и возможности записи. Администраторы тратят значительное время на обработку однотипных вопросов, что снижает эффективность работы клиники.

**Решение:** Интеллектуальный AI-ассистент, который:
- Автоматически обрабатывает запросы клиентов на естественном языке
- Ищет релевантную информацию о процедурах в базе данных
- Предоставляет персонализированные ответы с учетом контекста запроса
- Записывает клиентов на процедуры (в данном ноутбуке реализация не продемонстрирована)

**Бизнес-ценность:**
- Снижение нагрузки на администраторов
- Круглосуточная доступность информации для клиентов
- Мгновенная обработка запросов без ожидания
- Персонализированные рекомендации на основе актуальной базы данных
- Автоматизация процесса записи на процедуры

## Что происходит в ноутбуке

Демонстрационный ноутбук показывает базовый workflow работы RAG-системы:

### 1. Загрузка и валидация данных
- Чтение данных о процедурах из Excel-файла (`test.xlsx`)
- Валидация типов данных (строки, целые числа)
- Подготовка данных для загрузки в базу данных

### 2. Создание структуры базы данных
- Создание схемы `medical` в PostgreSQL
- Создание таблицы `procedures` с информацией о процедурах
- Создание таблицы `appointments` для записей клиентов

### 3. Настройка RAG-системы
- Инициализация LLM (в ноутбуке используется AWS Bedrock)
- Настройка ретривера, создающего SQL-запросы с помощью LLM
- Конфигурация промпта для генерации SQL с учетом особенностей данных (анестезия, стоимость)

### 4. Генерация ответа
- LLM формирует вежливый, структурированный ответ на основе найденных данных
- Предоставляет информацию о всех подходящих специалистах
- Предлагает записаться на процедуру


## Заключение

Данная демонстрационная версия показывает базовую концепцию работы RAG-системы для поиска медицинских процедур. Полная версия представляет собой готовое к production решение с диалоговым агентом, Telegram-ботом и автоматизированным ETL-пайплайном.
